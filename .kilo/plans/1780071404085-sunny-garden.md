# Fix DRAFT_STAGE_ERROR + Originality UI Visibility

## Root Cause Analysis

### Issue 1: DRAFT_STAGE_ERROR — `temperature` + `top_p` conflict
**Log evidence:** `400 {"type":"invalid_request_error","message":"\`temperature\` and \`top_p\` cannot both be specified for this model. Please use only one."}`

The Anthropic Sonnet 4.5 model rejects requests that specify both `temperature` and `top_p` simultaneously. The problem is in:
- `draftService.js` — `resolveSamplingParams()` returns both `temperature` AND `topP` for every tone, and both are passed to `useTool()`
- `twoPassDraftService.js` — same issue
- `anthropicClient.js` — `useTool()` passes both `temperature` and `top_p` to the SDK when both are defined

**Fix:** Remove `topP` from the tone sampling map. Only use `temperature`. The `topP` parameter should be removed from `useTool()` calls in draft services, or the `resolveSamplingParams` should only return temperature.

### Issue 2: No Originality/Copyright Check UI in Frontend
The wizard stepper shows only 5 stages: `research → outline → draft → seo → publish`. There is **no separate "Originality" step** in the frontend. The originality check runs **inside the draft stage** (in `wizardOrchestrator.js` after draft generation, before persistence) but:
1. The frontend has no visual indicator that originality checking is happening
2. The `DraftStep.jsx` component doesn't show any originality status, score, or progress
3. The `WizardStepper.jsx` has no originality stage at all
4. When `HLP_ORIGINALITY_GATE_ENABLED=true`, the check runs silently — if it fails, the draft stage fails with `ORIGINALITY_THRESHOLD_EXCEEDED` but the user sees no originality-specific UI

### Issue 3: Two-Pass Drafting Timeout Risk
With `HLP_TWO_PASS_DRAFTING_ENABLED=true`, the draft stage makes TWO LLM calls (Haiku rough → Sonnet polish). The draft timeout is 300,000ms (5 min). Two calls plus all validators may exceed this budget, causing `STAGE_TIMEOUT`.

## Plan

### Step 1: Fix temperature/top_p conflict (Critical)
**Files:** `backend/src/services/article/draftService.js`, `backend/src/services/article/twoPassDraftService.js`

- Remove `topP` from `TONE_SAMPLING` object — keep only `temperature`
- Update `resolveSamplingParams()` to return only `{ temperature }` 
- Remove `topP` from `useTool()` calls in both draft services
- The `anthropicClient.js` already handles `topP` conditionally — no change needed there

### Step 2: Add Originality Check UI to Draft Step
**Files:** `frontend/src/components/wizard/draft/DraftStep.jsx`, `frontend/src/redux/slice/wizard-slice.js`

- Add originality status display to `DraftStep.jsx`:
  - Show "Checking originality…" spinner when draft stage is running and originality gate is enabled
  - Show originality score/badge when draft is awaiting approval (pulled from `article.originalityRecord`)
  - Show originality failure message when draft fails with `ORIGINALITY_THRESHOLD_EXCEEDED`
- Add `originalityRecord` and `originalityScore` to `wizardSlice.js` draft state
- Wire `persistDraftOutput` data into the slice via `stageCompleted` output

### Step 3: Add Originality Stage to Wizard Stepper (Optional Visual Enhancement)
**Files:** `frontend/src/components/wizard/WizardStepper.jsx`

- Add "Originality" as a visual sub-step between Draft and SEO in the stepper
- This is purely visual — the backend still runs originality inside the draft stage
- Shows a spinner/check icon when originality is being checked/passed

### Step 4: Increase Draft Stage Timeout
**File:** `backend/src/constants/articleStatus.js`

- Increase `STAGE_TIMEOUT_MS.draft` from `300_000` (5 min) to `480_000` (8 min) to accommodate two-pass drafting + all validators

### Step 5: Add Better Error Messaging for Draft Failures
**Files:** `frontend/src/components/wizard/draft/DraftStep.jsx`

- Map `DRAFT_STAGE_ERROR` to more user-friendly messages based on the underlying `err.details`
- Show specific validator failure reasons (burstiness, paragraph variation, clichés, opening pattern) in the failure banner

## Implementation Order
1. Fix temperature/top_p conflict (immediate — blocks all draft generation)
2. Increase draft timeout (quick)
3. Add originality UI to DraftStep
4. Add originality to wizard slice state
5. Enhance error messaging

## Tradeoffs
- Removing `topP` means losing the per-tone top_p tuning. Alternative: only send `temperature` OR `top_p` (not both) — but Anthropic's SDK requires at least one. Temperature-only is the safest approach.
- Adding originality UI inside DraftStep (not as separate stage) matches the backend architecture where originality runs within the draft stage. Making it a separate stage would require backend refactoring.
